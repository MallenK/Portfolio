Checklist de rendimiento para webs en producción

19 septiembre 2025 · Lectura: 3 min

Este checklist resume los pasos que sigo antes de lanzar o optimizar un sitio en producción. Enfócate en métricas reales, automatiza auditorías y documenta cada ajuste.

1. Audita Core Web Vitals en contexto real

  • Analiza datos de CrUX o de PageSpeed Insights filtrando por país y dispositivo.
  • Configura performance budgets para LCP, FID y CLS.
  • Usa Lighthouse CI en cada PR y compara tendencias con WebPageTest.

2. Optimiza recursos críticos

El objetivo es entregar lo mínimo imprescindible para pintar el LCP. El resto debe cargarse diferido.

  • Prioriza CSS crítico inline y retrasa librerías pesadas (media="print" onload="this.media='all'").
  • Convierte imágenes a WebP/AVIF con múltiples tamaños y srcset.
  • Habilita loading="lazy" en iframes e imágenes no críticas.
  • Activa HTTP/2 push o preload selectivo para fuentes y scripts esenciales.

3. Minimiza JavaScript y controla el main thread

  1. Divide bundles con code-splitting y carga condicional según rutas.
  2. Reduce dependencias: revisa paquetes con npm ls y elimina polyfills innecesarios.
  3. Monitoriza el thread principal con Performance API y registra trazas en Analytics.

4. Cache y backend alineados

  • Define headers Cache-Control y ETag consistentes para estáticos y HTML.
  • Implementa CDN con stale-while-revalidate y edge functions para personalización ligera.
  • Revisa consultas lentas en MySQL/PostgreSQL con EXPLAIN y añade índices concretos.

5. Monitorización continua

El rendimiento se degrada con el tiempo. Programa revisiones mensuales con datos reales.

  • Instala Real User Monitoring (New Relic, Datadog, Sentry) y define alertas.
  • Documenta ajustes en un changelog de rendimiento.
  • Incluye KPIs de velocidad en los OKR del producto.
Pon en marcha esta checklist con tu equipo y medirás mejoras reales en tus conversiones. Escríbeme si necesitas una auditoría personalizada.
